# Deployment Checklist

Use this checklist to ensure everything is set up correctly before launching.

## Pre-Launch Setup

### GitHub Pages Setup
- [ ] Repository created on GitHub
- [ ] All files committed and pushed to `main` branch
- [ ] GitHub Pages enabled (Settings â†’ Pages)
- [ ] Site accessible at `https://YOUR_USERNAME.github.io/prompt_sharing/`
- [ ] Tested on desktop browser
- [ ] Tested on mobile device

### Google Form Setup
- [ ] Form created with all required fields (see `GOOGLE_FORM_SETUP.md`)
- [ ] Form description added
- [ ] Confirmation message configured
- [ ] Email notifications enabled
- [ ] Form tested with a sample submission
- [ ] Form URL copied for website

### Google Sheet Setup
- [ ] Sheet automatically created from form responses
- [ ] Sheet renamed to "SAIL Prompt Library"
- [ ] Headers formatted (bold, colored)
- [ ] Sharing permissions set:
  - [ ] Team members have editor access
  - [ ] Public link created (view-only)
- [ ] Sheet URL copied for website

### Website Updates
- [ ] Google Sheet URL added to `index.html` (line ~200, `id="sheet-link"`)
- [ ] Google Form URL added to `index.html` (line ~230, `id="form-link"`)
- [ ] Case studies customized with real examples (if available)
- [ ] Contact email updated in footer (if needed)
- [ ] Canvas link verified in footer

### Content Review
- [ ] Case study examples are relevant and helpful
- [ ] All text is clear and professional
- [ ] No placeholder text remains
- [ ] Links work correctly
- [ ] Navigation is smooth

## Testing

### Functionality
- [ ] All navigation links work
- [ ] Smooth scrolling works
- [ ] Google Sheet link opens correctly
- [ ] Google Form link opens correctly
- [ ] Form submission works
- [ ] Form responses appear in Sheet
- [ ] Email notifications received

### Design
- [ ] Site looks good on desktop (1920px, 1366px)
- [ ] Site looks good on tablet (768px)
- [ ] Site looks good on mobile (375px)
- [ ] Colors match Suffolk branding (if applicable)
- [ ] Fonts are readable
- [ ] Images/icons display correctly

### Content
- [ ] No typos or grammatical errors
- [ ] All instructions are clear
- [ ] Contact information is correct
- [ ] Links to Canvas/resources work

## Launch Preparation

### Communication
- [ ] Announcement drafted for Canvas
- [ ] Email to faculty prepared (if applicable)
- [ ] SAIL Friday session plan includes site introduction
- [ ] Support contact information ready

### Documentation
- [ ] README.md reviewed
- [ ] GOOGLE_FORM_SETUP.md reviewed
- [ ] Team members have access to documentation
- [ ] Maintenance workflow understood

## Post-Launch (First Week)

- [ ] Monitor form submissions daily
- [ ] Check for any technical issues
- [ ] Gather initial feedback
- [ ] Respond to questions promptly
- [ ] Review first submissions for quality

## Post-Launch (Ongoing)

- [ ] Weekly review of submissions (15-30 min)
- [ ] Monthly selection of featured prompts (30-60 min)
- [ ] Quarterly site updates with new case studies
- [ ] Annual review of form fields and structure

---

**Ready to launch?** Once all items are checked, share the site URL with faculty and start collecting prompts!
